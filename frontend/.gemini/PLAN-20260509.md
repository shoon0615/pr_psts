# FormSelect 빈 데이터 및 초기화 이슈 해결 계획

## 1. 문제 분석
하드코딩된 데이터임에도 불구하고 FormSelect가 빈 값으로 시작하거나 Placeholder가 보이지 않는 현상은 다음 두 가지 이유로 판단됩니다.

1.  **폼 상태 동기화 부재:** useForm은 초기 렌더링 시점의 defaultValues만 기억합니다. nuqs 등을 통해 URL 파라미터가 나중에 확정되거나 변경되어도 폼 내부의 field.value가 업데이트되지 않아 Placeholder와 데이터가 어긋납니다.
2.  **Select Value 매칭 오류:** Radix UI의 Select 컴포넌트는 value가 빈 문자열("")일 경우, 해당 값을 가진 SelectItem을 찾으려 시도합니다. 만약 일치하는 항목이 없으면 Placeholder를 보여주는 대신 아무것도 렌더링하지 않아 "빈 칸"으로 보이게 됩니다.

## 2. 해결 방안

### 단계 1: Form 컴포넌트 동기화 로직 추가 (shared/components/ui/custom/form.tsx)
- options.defaultValues가 변경될 때 form.reset()을 호출하여 외부 상태(URL 파라미터 등)와 폼 내부 상태를 강제로 동기화합니다.

### 단계 2: FormSelect 값 처리 개선 (shared/components/ui/custom/form.tsx)
- value={field.value || undefined}로 수정하여, 값이 없거나 빈 문자열일 경우 명시적으로 undefined를 전달합니다. 이를 통해 Select 컴포넌트가 정상적으로 Placeholder를 표시하도록 유도합니다.
- 불필요한 리렌더링과 상태 유실을 방지하기 위해 Select 컴포넌트의 key={items?.length} 속성을 제거합니다.

### 단계 3: SnackSearch 컴포넌트 정리 (app/(default-layout)/(main)/snack/_components/search.tsx)
- 데이터 로딩 상태와 폼 렌더링 타이밍을 맞추기 위해, 기존 useSnackSearchOptions 대신 Suspense 기반의 useSnackSearchOptions2 사용을 권장합니다.

## 3. 예상 결과
- 페이지 진입 시 URL 파라미터에 따라 FormSelect 값이 즉시 채워지거나, 값이 없을 경우 정확히 Placeholder("- 선택 -")가 노출됩니다.

위 계획에 대해 확인 부탁드립니다. 승인해 주시면 바로 수정을 진행하겠습니다.
